# DailyBell 🔔

DailyBell (เดลี่เบลล์) is a Thai-language daily reminder app for iPhone.
Add the things you do every day — skincare, water, study, exercise, work,
sleep — and get a local notification at the right time. Dark, premium, minimal.

- **UI:** Thai
- **Style:** black / charcoal with a single gold accent
- **Bundle ID:** `com.dailybell.app`
- **IPA name:** `DailyBell.ipa`
- **Build:** SwiftUI + XcodeGen, unsigned IPA via GitHub Actions, signed/installed with KSign

## Features
- Add / edit (tap a row) / delete (swipe left) reminders
- Mark done — **done status resets automatically each day**
- Fields: title, time, optional note, notification on/off, repeat-daily on/off (default on)
- Today's reminders sorted by time, with a separate "เสร็จแล้ว" (done) section
- Local notifications via `UNUserNotificationCenter`, permission requested on first launch
- Thai warning banner if notification permission is denied (with a Settings shortcut)
- **ทดสอบแจ้งเตือน** (test notification) button — fires a test in ~5 seconds
- Local storage with `Codable` + `UserDefaults`

> during the build, so they're intentionally **not** in the repo.

# iPhone-only guide

Everything below can be done entirely from an iPhone.

## 1) Upload the project to GitHub

**Easiest (no extra app): GitHub website**
1. In Safari, go to `github.com`, sign in, tap **+ → New repository**. Name it
   `DailyBell`, keep it **Public** (free Actions minutes), tap **Create**.
2. On the empty repo page, tap **uploading an existing file**.
3. Unzip `DailyBell.zip` first (tap it in the **Files** app → it unzips to a
   `DailyBell` folder). Then on the GitHub upload page tap **choose your files**,
   open the unzipped folder, and select everything.
   - If the picker keeps folder structure, you're done — commit.
   - If it flattens or skips `Assets.xcassets`, use **step 2** below to rebuild it.
4. Tap **Commit changes**.

**Smoother alternative: the Working Copy app**
- Install **Working Copy** (free) from the App Store.
- Create the repo on GitHub first, then in Working Copy: **clone** it, import the
  unzipped files into the working tree, **commit**, and **push**. Working Copy
  preserves folders (including `.xcassets`) correctly.

## 2) Handle `Assets.xcassets` correctly from iPhone

`.xcassets` and `.appiconset` are just **folders**. Most uploads keep them intact.
But some iPhone upload paths treat them as a single package and either skip them
or upload them as one blob — which breaks the icon.

If your icon folder didn't upload as a real folder:
1. Open the **`iphone_upload_helper`** folder in this project.
2. Follow its `README.md` — it walks you through recreating
   `Assets.xcassets/Contents.json` and
   `Assets.xcassets/AppIcon.appiconset/Contents.json` on the GitHub website by
   typing the path with `/`, then uploading the loose PNGs.
3. The build reads `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` from
   `project.yml`, so once the folder exists correctly the icon is picked up.

> Tip: confirm on GitHub that you can browse into
> `Assets.xcassets/AppIcon.appiconset/` and see 9 PNGs + a `Contents.json`.
> If you can, the icon will build.

## 3) Run the GitHub Actions build

- The workflow runs automatically on every push to `main`.
- To run it manually: open the repo → **Actions** tab → **Build Unsigned IPA**
  → **Run workflow** → pick `main` → **Run workflow**.
- Wait for the green check (first run ~3–6 minutes; it installs XcodeGen).

## 4) Download the IPA artifact

1. Open the finished run under the **Actions** tab.
2. Scroll to **Artifacts** at the bottom.
3. Tap **DailyBell-ipa** to download. GitHub gives you a `.zip`.
4. In the **Files** app, tap the downloaded zip to unzip it — inside is
   **`DailyBell.ipa`**.

## 5) Sign and install with KSign

1. Install **KSign** on your iPhone and add your signing certificate +
   provisioning profile (or your Apple ID / cert as KSign instructs).
2. Open KSign → import **`DailyBell.ipa`** (from the Files app).
3. Select your certificate/profile and tap **Sign**.
4. Tap **Install**. Approve the install prompt.
5. First launch: tap **Allow** when DailyBell asks for notification permission.
   - If you tap "Don't Allow," the app shows a Thai warning banner with a
     shortcut to enable notifications in Settings.
6. Use **ทดสอบแจ้งเตือน** on the main screen to confirm notifications work.

## Notes
- Minimum iOS 16. Portrait, iPhone, forced dark mode.
- The build is intentionally **unsigned**; KSign does the signing on device.
- Want a different app icon later? Replace the PNGs in
  `Assets.xcassets/AppIcon.appiconset/` (same filenames/sizes) and push again.
